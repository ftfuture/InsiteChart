# 부록

## 1. 기본 용어집

### 1.1 기본 기술 용어

| 용어 | 영문 | 설명 |
|------|------|------|
| API | Application Programming Interface | 애플리케이션 프로그래밍 인터페이스. 소프트웨어 구성 요소들이 서로 통신할 수 있게 하는 규약과 도구의 집합 |
| CI/CD | Continuous Integration/Continuous Deployment | 지속적 통합/지속적 배포. 코드 변경 시 자동으로 빌드, 테스트, 배포를 수행하는 개발 방법론 |
| CORS | Cross-Origin Resource Sharing | 웹 페이지가 다른 출처(origin)의 리소스에 접근할 수 있도록 하는 브라우저 보안 기능 |
| CRUD | Create, Read, Update, Delete | 데이터베이스에서 기본적인 데이터 조작 기능 |
| DTO | Data Transfer Object | 데이터 전송 객체. 계층 간 데이터를 전달하기 위한 객체 |
| JWT | JSON Web Token | JSON 형식의 토큰 기반 인증 방식 |
| MVC | Model-View-Controller | 소프트웨어 아키텍처 패턴. 애플리케이션을 세 개의 구성 요소로 분리 |
| ORM | Object-Relational Mapping | 객체 관계 매핑. 객체 지향 프로그래밍 언어와 관계형 데이터베이스 간의 데이터 변환 기술 |
| PII | Personally Identifiable Information | 개인 식별 정보. 개인을 식별할 수 있는 정보 |
| REST | Representational State Transfer | 웹 아키텍처 제약 조건 집합. 자원을 URI로 식별하고 HTTP 메서드로 조작 |
| SDK | Software Development Kit | 소프트웨어 개발 키트. 특정 플랫폼이나 프레임워크용 개발 도구 모음 |
| SLA | Service Level Agreement | 서비스 수준 협약. 서비스 제공자와 이용자 간의 서비스 품질에 대한 합의 |
| SSO | Single Sign-On | 통합 인증. 하나의 인증 정보로 여러 시스템에 접근하는 방식 |
| TPS | Transactions Per Second | 초당 트랜잭션 수. 시스템 처리 능력 측정 지표 |
| UI/UX | User Interface/User Experience | 사용자 인터페이스/사용자 경험 |
| WAF | Web Application Firewall | 웹 애플리케이션 방화벽. 웹 공격을 탐지하고 차단하는 보안 장치 |

## 7. 기본 트러블슈팅 가이드

### 7.1 기본적인 문제 해결

#### 7.1.1 기본 로그인 문제

**증상**: 로그인이 안 되거나 계속 실패
**원인**:
- 잘못된 자격증명 입력
- 계정 잠금
- 세션 만료
- 네트워크 문제

**해결책**:
1. 자격증명 확인
2. 계정 잠금 상태 확인
3. 브라우저 캐시 삭제
4. 네트워크 연결 확인
5. 관리자에게 문의

#### 7.1.2 기본 데이터 로딩 문제

**증상**: 데이터가 로딩되지 않거나 느림
**원인**:
- API 서버 다운
- 네트워크 지연
- 데이터베이스 성능 문제
- 브라우저 호환성 문제

**해결책**:
1. API 서버 상태 확인
2. 네트워크 속도 테스트
3. 데이터베이스 쿼리 기본 최적화
4. 브라우저 업데이트
5. 캐시 삭제

#### 7.1.3 기본 검색 결과 없음

**증상**: 검색해도 결과가 나오지 않음
**원인**:
- 검색어 오타
- 검색 인덱스 문제
- 필터링 조건 너무 제한적
- 데이터 없음

**해결책**:
1. 검색어 철자 확인
2. 검색어 일부로 검색
3. 필터링 조건 완화
4. 검색 인덱스 재구축
5. 데이터 존재 여부 확인

### 7.2 기본 성능 문제 해결

#### 7.2.1 기본 페이지 로딩 속도 저하

**증상**: 페이지 로딩이 매우 느림
**원인**:

1. 이미지 기본 최적화 및 압축

4. 서버 성능 기본 향상
5. 브라우저 캐시 활용

**원인**:

1. 쿼리 기본 최적화 및 인덱스 추가
2. 데이터베이스 연결 풀 조정
3. 서버 사양 기본 증설
4. 기본 캐시 도입
5. API 비동기 처리

## 8. FAQ

### 8.1 일반적인 질문

**Q: 주식 데이터는 얼마나 자주 업데이트되나요?**
A: 실시간으로 업데이트됩니다. 시장이 열려 있는 동안에는 15초 간격으로 데이터를 수집하며, 시장이 마감되면 일일 종가로 업데이트됩니다.

**Q: 어떤 주식 시장을 지원하나요?**
A: 현재는 미국 주식 시장(NYSE, NASDAQ)을 지원하며, 향후 한국, 일본, 유럽 등 다른 시장으로 확장할 계획입니다.

**Q: 센티먼트 분석은 어떤 소셜 미디어를 기반으로 하나요?**
A: Reddit과 Twitter의 데이터를 기반으로 분석합니다. 향후 다른 소셜 미디어 플랫폼도 추가할 예정입니다.

**Q: 모바일 앱은 언제 출시되나요?**
A: 현재 모바일 웹 버전을 지원하며, iOS 및 Android 네이티브 앱은 2024년 3분기 중 출시를 목표로 개발 중입니다.

**Q: 실시간 알림은 어떻게 받을 수 있나요?**
A: 웹 푸시 알림, 이메일 알림, SMS 알림을 지원합니다. 설정에서 알림 종류와 수신 시간을 조정할 수 있습니다.

### 8.2 기술적인 질문

**Q: 어떤 기술 스택을 사용하나요?**
A: 프론트엔드는 React와 TypeScript를, 백엔드는 Python과 FastAPI를, 데이터베이스는 PostgreSQL과 TimescaleDB를 사용합니다.

**Q: API 속도 제한이 있나요?**
A: 인증된 사용자는 시간당 1000회, 인증되지 않은 사용자는 시간당 100회의 요청을 할 수 있습니다. 유료 플랜에서는 제한이 완화됩니다.

**Q: 데이터는 어떻게 보호되나요?**
A: 전송 중에는 HTTPS로 암호화되며, 저장 중에는 암호화하여 보관합니다. 또한 정기적인 백업과 접근 제어를 통해 데이터를 보호합니다.

**Q: 시스템 장애 시 어떻게 대응하나요?**
