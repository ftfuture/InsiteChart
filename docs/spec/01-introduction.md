# 프로젝트 개요 및 목표

## 1. 프로젝트 배경

### 1.1 현황 분석
현재 주식 차트 분석 애플리케이션은 기본적인 차트 시각화와 간단한 검색 기능을 제공하고 있습니다. 하지만 현대적인 투자자들이 요구하는 고급 기능과 사용자 경험에는 부족한 상황입니다.

### 1.2 시장 요구사항
- **실시간 데이터**: 투자 결정에 필요한 최신 정보 제공
- **소셜 인사이트**: 커뮤니티의 여론과 트렌드 분석
- **개인화된 경험**: 사용자별 맞춤형 정보 제공
- **접근성**: 모든 사용자가 쉽게 접근할 수 있는 인터페이스
- **확장성**: 증가하는 사용자 트래픽을 감당할 수 있는 아키텍처

## 2. 프로젝트 목표

### 2.1 비전
"데이터 기반의 스마트한 투자 결정을 지원하는 종합 금융 분석 플랫폼 구축"

### 2.2 핵심 목표
1. **고급 검색 기능**: 자동완성, 필터링, 검색 히스토리
2. **소셜 센티먼트 분석**: Reddit, Twitter 등 커뮤니티 데이터 통합
3. **실시간 트렌딩**: 언급량 급증 주식 자동 감지
4. **개인화된 추천**: 사용자 행동 기반 맞춤형 정보 제공
5. **확장 가능한 아키텍처**: 마이크로서비스 기반의 확장성 확보

### 2.3 사용자 요구사항 상세

#### 2.3.1 Enhanced Stock Search 요구사항
1. **실시간 자동완성**
   - 사용자가 2글자 이상 입력하면 관련 주식 목록 실시간 표시
   - 심볼명과 회사명 모두에서 부분 일치 검색 지원
   - 관련도 순으로 정렬된 최대 10개 검색 제안 표시

2. **검색 히스토리 관리**
   - 사용자 검색 기록 자동 저장 (최근 20개 유지)
   - 검색창 클릭 시 최근 검색 기록 드롭다운 표시
   - 중복 제거 및 최신 검색 시간 기준 정렬

3. **다차원 필터링**
   - 주식 유형(EQUITY, ETF, MUTUALFUND, INDEX) 필터링
   - 섹터별 및 거래소별 필터링
   - 여러 필터 동시 적용 및 실시간 결과 업데이트

4. **관심종목 관리**
   - 카테고리별 그룹화 및 드래그 앤 드롭 순서 변경
   - 개인 메모 추가 및 중복 방지
   - 로컬 스토리지 자동 저장

5. **성능 및 안정성**
   - 500ms 이내 검색 응답 시간
   - 네트워크 오류 시 적절한 에러 메시지 표시
   - 검색 결과 캐싱 및 동시 요청 안정적 처리

#### 2.3.2 Social Sentiment Tracker 요구사항
1. **실시간 언급 추적**
   - 실시간 주식 언급 데이터 수집 및 표시
   - 1시간, 24시간, 7일 단위 언급 횟수 집계
   - 상위 20개 언급 주식 랭킹 형태 표시
   - 5분 간격 자동 데이터 업데이트

2. **트렌딩 감지**
   - 24시간 대비 언급량 200% 이상 증가 주식 감지
   - 트렌딩 주식 별도 섹션 우선 표시
   - 시각적 알림 표시 및 트렌딩 시작 시간/증가율 표시
   - 언급량 변화 그래프 시각화

3. **커뮤니티 필터링**
   - 단타, 가치투자, 성장투자 카테고리로 커뮤니티 분류
   - 사용자 투자 성향 선택 및 여러 성향 동시 선택 지원
   - 커뮤니티별 언급 비중 시각적 표시

4. **센티먼트 분석**
   - 각 언급의 긍정, 중립, 부정 감정 분석
   - -100에서 +100 범위의 주식별 전체 감정 점수 표시
   - 감정 변화 추이 시간별 추적 및 색상 시각화
   - 상세 감정 분석 결과 표시

5. **차트 통합**
   - 주식 차트 위에 언급량 데이터 오버레이 표시
   - 감정 점수 변화 별도 서브플롯 표시
   - 특정 시점 선택 시 해당 시점 주요 언급 내용 표시
   - 언급량 급증 시점 마커 표시 및 상관관계 지표 계산

## 3. 프로젝트 범위

### 3.1 포함 기능
- **Enhanced Stock Search**
  - 실시간 자동완성 검색
  - 다차원 필터링 (유형, 섹터, 거래소 등)
  - 검색 히스토리 및 관심종목 관리
  - 관련도 기반 검색 결과 정렬

- **Social Sentiment Tracker**
  - 다양한 소셜 미디어 데이터 수집
  - 실시간 센티먼트 분석 및 시각화
  - 트렌딩 주식 감지 및 알림
  - 커뮤니티별 분석 및 필터링

- **통합 시스템**
  - 검색 결과에 센티먼트 정보 통합
  - 관심종목 실시간 센티먼트 상태 표시
  - 차트에 소셜 데이터 오버레이

### 3.2 제외 기능
- 실시간 거래 실행 기능
- 금융 자문가 추천
- 포트폴리오 관리 자동화
- 세금 보고 기능

## 4. 성공 지표

### 4.1 기술적 지표
- **성능**: API 응답 시간 500ms 이하
- **가용성**: 99.9% 이상의 서비스 가용성
- **확장성**: 10,000 동시 사용자 지원
- **정확도**: 센티먼트 분석 정확도 85% 이상

### 4.2 비즈니스 지표
- **사용자 참여도**: 일일 활성 사용자 30% 증가
- **기능 사용률**: 신규 기능 사용률 60% 이상
- **사용자 만족도**: 사용자 만족도 점수 4.5/5.0 이상
- **데이터 커버리지**: 상위 500 주식 데이터 완전한 커버리지

## 6. 제약 조건

### 6.1 기술적 제약
- **프로그래밍 언어**: Python 3.11+
- **프레임워크**: Streamlit, FastAPI
- **데이터베이스**: PostgreSQL, TimescaleDB
- **클라우드**: AWS 기반 인프라
- **API**: RESTful API 설계

### 6.2 비즈니스적 제약
- **예산**: 정해진 예산 범위 내 개발
- **일정**: 8개월 내 프로젝트 완료
- **규제**: 금융 관련 규제 준수
- **보안**: 개인정보 보호 및 데이터 보안

## 7. 위험 요소

### 7.1 기술적 위험
- **API 의존성**: 외부 API 변경에 따른 영향
- **데이터 품질**: 소셜 데이터의 신뢰성 문제
- **성능 병목**: 대량 데이터 처리의 성능 저하
- **확장성**: 예상보다 빠른 사용자 증가

### 7.2 비즈니스적 위험
- **경쟁**: 유사 서비스와의 경쟁 심화
- **규제 변경**: 금융 규제 강화에 따른 영향
- **사용자 채택**: 신규 기능의 사용자 채택 부진
- **데이터 비용**: 외부 데이터 수집 비용 증가
